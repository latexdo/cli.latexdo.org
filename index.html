Terminal install

Install the local editor with one command.

The CLI keeps a cached copy of the LatexDo source, installs npm dependencies when they change, starts the local web editor, and opens the localhost URL in your browser.

curl -fsSL https://cli.latexdo.org/install.sh | bash
View install script CLI source

Local workflow

Run LatexDo from your terminal.

The latexdo command is the terminal entry point for launching, updating, checking, and resetting the local LatexDo app.

Cached source

Keeps the LatexDo app in a local cache so repeated launches are fast.

Dependency checks

Installs npm dependencies only when the project dependencies change.

Localhost editor

Starts the web editor locally and opens the browser URL for you.

Portable config

Uses environment variables for cache location, source repo, branch, host, and port.

Commands

Everything the CLI can do.

latexdo

Default command. Starts the local editor, updates cached source, installs dependencies when needed, and opens the browser.

latexdo open

Same as the default command. latexdo serve is also accepted.

latexdo open --host 127.0.0.1 --port 5173

Choose the bind host and preferred port. If the port is busy, LatexDo uses the next free port.

latexdo open --no-update

Start from the cached app without fetching the latest source first.

latexdo update

Fetch the configured app branch and install npm dependencies without starting the editor.

latexdo doctor

Print cache paths, repo settings, Git, Node.js, npm, curl, and latexmk status.

latexdo path

Print the local cached app directory.

latexdo reset

Remove the cached LatexDo app so the next launch performs a clean clone.

latexdo help

Show command usage and environment options.

Requirements

Git, Node.js 20+, and npm.

The CLI uses Git to cache the LatexDo source and npm to install and run the local web editor.

Required

Install Git, Node.js 20 or newer, npm, and a browser before running latexdo.

Installer download

The install script uses curl or wget to download the executable CLI script.

PDF compilation

Browser editing works without a TeX distribution, but local PDF compilation needs latexmk.

Configuration

Environment variables.

Defaults are designed for a normal local install. Override them when you need a custom cache, fork, branch, binary directory, or local port.

Variable Default Purpose
LATEXDO_HOME ~/.latexdo Base cache directory for CLI state and source.
LATEXDO_APP_DIR $LATEXDO_HOME/app Local Git checkout used by the editor.
LATEXDO_APP_REPO https://github.com/latexdo/latexdo.git Source repository cloned by the CLI.
LATEXDO_APP_BRANCH main Branch fetched and checked out during updates.
LATEXDO_HOST 127.0.0.1 Local host used by the Vite dev server.
LATEXDO_PORT 5173 Preferred local port. The CLI finds the next free port if needed.
LATEXDO_NO_OPEN 0 Set to 1 to print the localhost URL without opening a browser.
LATEXDO_SKIP_UPDATE 0 Set to 1 to start from cached source without fetching first.
LATEXDO_BIN_DIR ~/.local/bin Install location used by install.sh.
LATEXDO_CLI_URL https://latexdo.org/bin/latexdo CLI script URL downloaded by install.sh.
LATEXDO_SKIP_BOOTSTRAP 0 Set to 1 to install the CLI without immediately running latexdo update.

Install details

What the installer changes.

Binary location

The CLI is installed as ~/.local/bin/latexdo unless LATEXDO_BIN_DIR is set.

PATH check

If the install directory is not on PATH, the installer prints the shell export line to add.

Bootstrap

After install, it runs latexdo update to clone the app and install dependencies.

Troubleshooting

Common fixes.

latexdo is not found after installation

Add ~/.local/bin to your shell PATH, or install to a directory already on PATH with LATEXDO_BIN_DIR.

Node.js version error

Install Node.js 20 or newer, then run latexdo doctor to confirm the active runtime.

Port 5173 is already in use

The CLI automatically scans for the next free port. Use --port or LATEXDO_PORT to choose a different preferred port.

Cached source looks broken

Run latexdo reset, then run latexdo update or latexdo to clone a clean copy.

Local PDF compilation is unavailable

Install a TeX distribution that provides latexmk. The browser editor can still open without it.
